# 01 — 产品愿景与功能蓝图

## 项目定位

「飘悠番茄 Web」是一个独立的 **Web / PWA 番茄钟应用**。

**核心目标：**
- **极简专注体验**：界面克制，沉浸式专注模式
- **跨平台即用**：任意现代浏览器打开即用，PWA 可安装到桌面/主屏、离线可用
- **纯本地零追踪**：V1.0 无后端、无网络请求、无第三方 SDK，所有数据存 IndexedDB
- **流畅 Q 弹体验**：spring 物理过渡动画 + 多材质（毛玻璃/黏土等）UI，让交互有温度

### 为什么做

一个独立、跨平台、免安装、零追踪的极简专注工具。任意浏览器打开即用，PWA 可安装、可离线，体验接近原生却不依赖任何应用商店审核。

## V1.0 — 纯前端 PWA 版（首版上线）

### 计时核心
- 经典番茄循环（默认 25 / 5 / 15，4 轮长休）
- 自定义时长（工作 / 短休 / 长休 / 长休间隔）
- 多套番茄预设方案（经典 25/5、深度 50/10、马拉松 90/20，可增删）
- 暂停 / 恢复（单次暂停限时 5 分钟，用户可自定义 30s ~ 30min 或不限时；超时自动结束并标记中断）
- 跳过当前阶段
- 提前结束 / 放弃番茄
- **任务锁定**：番茄启动后（Working/Paused 阶段）当前 `taskId` 锁定，UI 禁用切换

### 任务管理
- 任务（名称 + 颜色）
- 番茄关联任务
- 任务归档

### 记录与统计
- 番茄记录持久化（IndexedDB）
- 今日 / 本周 / 本月 / 总计 完成数 + 专注时长
- 任务维度专注分布
- **专注热力图**（GitHub 风格年度格子图，主题色深浅 = 当日专注时长，长按/悬停查看明细）

### 通知与后台（Web 平台方案）
> Web 后台限制严格：浏览器切后台后 setTimeout/setInterval 会被节流甚至冻结，无法可靠后台计时。Web 版策略：
- **可见性 API 校准**：`visibilitychange` + `focus` 事件触发时，用 `Date.now()` 重算剩余时间
- **Notification API**：番茄完成时弹系统通知（需用户授权），点击通知回到页面
- **Service Worker 定时唤醒（受限）**：用 `notificationManager` 或 `setTimeout` 在 SW 内注册延迟通知，但浏览器对 SW 后台存活时间有限（Chrome ~5 分钟），仅作短时兜底
- **不依赖后台计时**：番茄进行中切后台，回来后用 `Date.now() - startAt` 校准，若已超时则按规则跳阶段或标记中断
- **屏幕常亮**：Wake Lock API（`navigator.wakeLock.request('screen')`），番茄进行中防熄屏

### 桌面入口（PWA）
- **可安装**：满足 PWA criteria 后，浏览器提示「添加到主屏/桌面」，安装后独立窗口运行
- **离线可用**：Service Worker 缓存 App Shell + IndexedDB 数据，断网仍可计时
- **无桌面小卡片**：Web 标准缺乏小卡片 API，用 PWA 图标 + 通知替代

### 自定义外观
- 8 套内置主题色
- 4 种数字样式（经典 / 细线 / 机械翻页 / 点阵数码）
- 卡片材质：扁平 / 毛玻璃（`backdrop-filter: blur`）/ 厚毛玻璃 / 云母（半透明 + 微噪声）/ 黏土 / 新拟态
- 4 套背景渐变 + 随机模式
- 深色模式三档：跟随系统 / 强制浅色 / 强制深色

### 沉浸专注模式
- 全屏时钟模式（Fullscreen API + 防烧屏微动 + 降亮 + 避纯白）
- 背景渐变轮播（每次进入随机一套，停留期间固定）

### UI / 动效基调（Q 弹流畅）
- **spring 物理过渡**：启停、状态切换、按压回弹走 spring（轻微 overshoot），非生硬线性
- **多材质体系**：毛玻璃 / 厚毛玻璃 / 云母 / 黏土 / 新拟态 / 扁平，`MaterialBox` 统一封装
- **微交互**：列表项 stagger 入场、模态从触发源 scale+fade、共享元素转场（圆环从首页缩放到全屏专注页）
- **降级**：`prefers-reduced-motion` + 低端设备降级为扁平材质 + 简化动画，保体验一致

### 通用
- 屏幕常亮（Wake Lock）、振动（Vibration API，移动端）、完成提示音（Web Audio API）
- **数据导出 JSON**（应用自有 schemaVersion，支持备份与跨设备迁移）
- **数据导入 JSON**：导入导出文件可还原数据，LWW 冲突解决
- 隐私协议、用户协议、首次引导
- **键盘快捷键**（Web 优势）：空格启停 / S 跳过 / F 全屏 / T 切任务 / , 设置
- **响应式一多**：CSS 媒体查询 + Tailwind 断点，手机/平板/PC 一套代码

### V1.0 纯净版策略
- **无后端、无网络请求、无第三方 SDK**（不接统计 / 推送 / 崩溃上报 / 广告）
- 所有数据存 IndexedDB + localStorage，卸载即消失
- 隐私声明可勾「不收集任何用户个人信息」
- 托管在境外（Vercel / Cloudflare Pages），免 ICP 备案

## V1.1 — 本地体验深化（下一站）

> V1.0 上线后真用户反馈显示：核心流程已稳，但**质感层缺位**。V1.1 不引入后端，全部在现有 PWA + IndexedDB 架构内深化。
>
> 配方目标：5-7 个工作日落地，每一项可量化、可独立交付、不破零追踪。

### 1. 完成音效 + 振动落地（半天）
- **背景**：[SettingsPage.tsx](../src/pages/SettingsPage.tsx) 已留 TODO：「音效 / 振动 / 暂停限时 等设置项后续阶段落地」。V1.0 内置 [AudioService](../src/service/AudioService.ts) 但未接到 UI 设置开关。
- **落地点**：番茄结束 → 提示音 / 移动端振动；设置页 4 档音量 + 静音开关 + 振动开关 + 试听按钮。
- **数据**：扩 `appearanceStore` 或新建 `preferencesStore` 持久化（localStorage）。

### 2. 白噪音 / 专注音轨（2-3 天）—— 从 V1.2 拆过来
- **价值**：带耳机专注是常态，白噪音是真用户高频痛点。
- **MVP**：4-6 段内置音轨（雨声 / 咖啡馆 / 森林 / 海浪 / 白噪音纯色 / 粉噪音），HTML5 `<audio>` 循环播放 + 音量滑块。
- **范围限缩**：暂不做用户上传 / 多轨混音 / 频谱可视化（留 V1.2 完整版）。
- **数据**：音频文件作 PWA precache 静态资源（每段 ~200KB 压缩，6 段 ~1.2MB，可接受）。
- **离线**：Service Worker 预缓存音轨，断网仍可播。

### 3. 统计详情时间线（1-2 天）
- **现状**：[StatsPage](../src/pages/StatsPage.tsx) 仅有聚合视图（今日/本周/本月 + 热力图 + 任务分布）。
- **缺**：单条番茄记录回顾。用户想知道「上周二专注了什么任务、几个番茄、什么时段」无入口。
- **落地**：StatsPage 加「最近 N 条记录」时间线（默认 20 条，可加载更多）；每条显示任务名 / 时长 / 起止时间 / 是否完成。
- **架构**：直接读 [SessionDao](../src/service/SessionDao.ts) 现有查询能力，UI 层加一个时间线组件。

### 4. 极克制成就系统（1-2 天）
- **价值**：留存利器，但有"小红书化"风险（积分、排行榜、社交压力）。
- **克制原则**：
  - 只做**里程碑成就**（首次完成 / 累计 10 / 50 / 100 个番茄 / 7 天连续 / 30 天累计 100 小时），固定 ~8 个，不做积分等级
  - **不做排行榜**（违背单机零追踪定位）
  - **不做推送弹窗**，仅在「关于」上方新增"成就墙"独立 tab 入口被动展示
  - 已解锁徽章可点击查看解锁日期 + 解锁条件
- **数据**：成就解锁状态存 IndexedDB（新建 `achievements` 表），每次番茄完成后 hook 检查解锁逻辑。
- **风险**：若用户反馈"打扰专注"，整个 V1.1 可一键关闭（设置页开关）。

### V1.1 不做的
- 番茄日记 / 心情记录 → V1.2（侵入流程，需要更多产品打磨）
- 自定义快捷键 → V2.0（重度用户向，V1.1 优先大众痒点）
- 用户上传自定义音轨 → V1.2 完整音频版
- 任何后端 / 账号 / 同步 → V2.x（路线决策见 [10-decisions-log.md](10-decisions-log.md) 2026-06-23 条目）

## V1.2 — 音频深化 + 番茄日记

> V1.1 已落地白噪音 MVP，本版做完整音频生态 + 把番茄从纯计时升级为反思工具。

- **用户上传本地音频** 作白噪音（IndexedDB blob，~5MB 上限）
- **多轨混音**：内置音轨可叠加，独立音量调节
- **真实音频频谱**（Web Audio AnalyserNode），全屏专注页底部叠加可视化
- **番茄日记**：每个番茄结束后可选写两句感想 + 心情标签（5 档表情），可关闭
- 数字样式从 4 种扩到 6+ 种（用户呼声多再扩）

## V2.0 — 高级自定义

> 此处开始考虑面向重度用户的能力开放。

- 用户上传背景图 / 自定义提示音
- **自定义快捷键映射**（把当前 HOTKEYS 常量改为可持久化 store + 设置页 UI）

## V2.x — 云同步版（按用户反馈触发）

> **启动条件**（[10-decisions-log.md](10-decisions-log.md) 2026-06-23 条目定）：本地版部署后跑一段时间 + 真实用户提出过同步诉求。在此之前不动相关代码。
>
> **形态**：运行时插件（`VITE_SYNC_ENDPOINT` 环境变量分叉），不开双 build target、不开双 git 分支。
>
> **范围**：多用户账号 + 跨设备同步。不做实时协作，不做云端独占功能（避免本地版降级为「免费试用版」）。

- 账号体系（候选：Magic link 邮件 / OAuth GitHub）
- 同步协议：基于 V1.0 已有的 LWW 冲突解决，扩到 server-side
- 自部署友好：用户用自己的服务器跑同步后端也可
